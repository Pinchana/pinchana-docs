## Scrape request

| Field | Type | Required | Meaning |
| --- | --- | ---: | --- |
| `url` | HTTP(S) URL | yes | Complete supported source URL. Validation failures return `422`. |

```json
{"url":"https://www.instagram.com/p/SHORTCODE/"}
```

## Success envelope

API v1 returns `data` plus version metadata:

```json
{
  "data": {
    "id": "SHORTCODE",
    "source": {
      "platform": "instagram",
      "url": "https://www.instagram.com/p/SHORTCODE/",
      "application": null
    },
    "content": {
      "title": null,
      "text": "Example post",
      "html": null,
      "published_at": "2026-07-15T11:56:00Z"
    },
    "author": {"name": "Creator", "username": "creator"},
    "media": [],
    "music": null,
    "engagement": null,
    "safety": null,
    "link": null
  },
  "meta": {"api_version": "1"}
}
```

| Object | Fields | Notes |
| --- | --- | --- |
| `source` | `platform`, `url`, `application` | `platform` is one of the configured module identifiers. `application` identifies a source application when supplied. |
| `content` | `title`, `text`, `html`, `published_at` | Every value is nullable. Timestamps use an ISO 8601 representation. |
| `author` | `name`, `username` | Both values are nullable because upstream platforms expose different identity fields. |
| `music` | `album` | Present only when album metadata exists outside individual media assets. |
| `engagement` | `likes`, `replies`, `reposts`, `quotes`, `views` | Counts are nullable and non-negative. Absence does not mean zero. |
| `safety` | `spoiler`, `text_spoiler`, `nsfw` | Boolean display and content-safety signals. |
| `link` | `url` | Optional extracted external link. |

## Media assets

All downloadable or previewable files appear in one ordered `media` array.

| Field | Type | Meaning |
| --- | --- | --- |
| `index` | non-negative integer | Stable order within this response |
| `type` | `image`, `video`, or `audio` | Media transport type |
| `role` | `content`, `soundtrack`, or `cover` | Semantic purpose of the file |
| `url` | string | Authenticated relative media path |
| `preview_url` | string or null | Preview or thumbnail path, usually for video |
| `dimensions` | object or null | Positive `width` and `height` measured from cached visual media |
| `duration_seconds` | integer or null | Non-negative duration when known |
| `title`, `artist` | string or null | Asset-level music metadata |
| `looping` | boolean | Indicates video intended to repeat, such as Twitter/X GIF-style media |

Text-only posts are valid and return an empty `media` array. Clients must not treat an empty array as a schema failure.

## Error envelope

API v1 uses one error shape for HTTP, validation, routing, and upstream failures:

```json
{
  "error": {
    "code": "authentication_required",
    "message": "This media requires authentication",
    "details": null
  }
}
```

`details` may contain structured validation information. Clients should branch on `code`, display a controlled local message, and treat the server message as diagnostic text.

## Legacy compatibility

The unversioned `/scrape` and `/web/scrape` routes return the earlier flat `ScrapeResponse` with fields such as `shortcode`, `media_type`, `thumbnail_url`, `video_url`, `carousel`, and `tracklist`.

<Aside type="note">
Legacy routes remain supported for existing clients. New integrations must use API v1. The normalized contract prevents platform-specific fields from becoming client dependencies.
</Aside>
