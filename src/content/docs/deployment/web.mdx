Pinchana Web requires a running server. It cannot be exported as static HTML because route handlers proxy authenticated requests, verify Ed25519 certificates, read server-only configuration, and manage HttpOnly cookies.

## Environment split

| Variable | Phase | Exposure |
| --- | --- | --- |
| `NEXT_PUBLIC_TURNSTILE_SITE_KEY` | build/public | Embedded for the browser; register it for the production Web hostname |
| `PINCHANA_API_URL` | runtime/server | Default API origin; never public |
| `PINCHANA_INSTANCE_PUBLIC_KEY` | runtime/server | Ed25519 public verification PEM; preserve newlines or use escaped `\n` |

Configure the selected API separately with the matching Turnstile secret, expected Web hostname `pinchana.example.com`, action `turnstile-spin-v1`, and signed-session secret.

<Tabs>
<TabItem label="Vercel">

1. Import `Pinchana/pinchana-web` as a Next.js project.
2. Use Bun according to the repository lockfile and keep install/build scripts enabled so FFmpeg assets synchronize.
3. Add the three environment variables to Production (and suitable test values to Preview/Development).
4. Deploy, then confirm `public/ffmpeg/ffmpeg-core.wasm` and worker assets are served.
5. Register the final Vercel/custom hostname in Turnstile and set the API's expected hostname exactly.

`NEXT_PUBLIC_...` changes require a rebuild. Server-only values should be available to route handlers at runtime; redeploy after changing them so all instances receive consistent configuration.

</TabItem>
<TabItem label="Node or Bun host">

```sh
git clone https://github.com/Pinchana/pinchana-web.git
cd pinchana-web
bun install --frozen-lockfile
bun run build
bun run start
```

Supply the environment through your service manager, run behind an HTTPS reverse proxy, and keep the process supervised. The default Next start port is 3000 unless overridden through normal Next runtime options. Forward streaming responses and byte ranges without buffering the entire media file.

</TabItem>
</Tabs>

## Verification checklist

- Page load checks an existing session or renders Turnstile.
- Browser network requests never reveal the API origin, signed session, or instance certificate cookie.
- URL processing, image/video/audio preview, byte-range seeking, Media and Audio-only saves, ZIP creation, and FFmpeg loading work.
- Cookies are `HttpOnly`, `SameSite=Strict`, and `Secure` over production HTTPS.
- Custom origin selection accepts an authorized certificate and rejects mismatched/expired envelopes.

<Aside type="caution">
Do not add a machine `PINCHANA_API_KEYS` secret to Pinchana Web. Its browser flow is intentionally isolated and uses the API's signed `/web/...` session routes.
</Aside>
