## Settings

The `pinchana-settings` local-storage object is updated when preferences change:

```json
{
  "autoSave": true,
  "zipMultiple": true,
  "filenameStyle": "pretty",
  "pawsEnabled": true,
  "reduceMotion": false,
  "convertTwitterGifs": true,
  "downloadMode": "media",
  "dlpQuality": "best",
  "dlpCodec": "auto",
  "dlpContainer": "auto",
  "dlpAudioFormat": "mp3",
  "dlpAudioBitrate": "128",
  "preferBetterAudio": false,
  "dubLanguage": "original",
  "subtitleLanguage": "none"
}
```

| Preference | Default | Effect |
| --- | ---: | --- |
| Save immediately | on | Starts the save after processing/conversion |
| ZIP multiple files | on | Combines carousel or track-list files into one archive |
| Filename style | Pretty | Chooses Classic, Basic, Pretty, or Nerdy branded filenames; every style keeps `[pinchana.cc]` |
| Background paws | on | Shows the decorative floating pattern |
| Reduce motion | off | Disables interface animations/transitions |
| Convert Twitter GIFs | on | Converts looping Twitter/X video assets to GIF when saving; disabling it keeps the source video |
| Download mode | Media | Remembers Media or Audio only |
| YouTube video options | Best / Auto / Auto | Preferred quality, codec, and container |
| YouTube audio format | MP3 | Best source, MP3, OGG, WAV, or Opus |
| YouTube audio bitrate | 128 kb/s | Used for lossy audio conversion |
| Better YouTube audio | off | Prefers higher-bitrate/sample-rate audio streams |
| Preferred dubbed track | Original | Uses the selected language when available |
| Embedded subtitles | None | Embeds the preferred creator subtitle, falling back to automatic captions, in YouTube video downloads only |

The **YouTube** settings category also contains the encrypted Cookie Vault. Vault labels, domains, and cookie values remain inside its separately encrypted IndexedDB payload; they are never stored in `pinchana-settings`.

The custom API origin is deliberately **not** kept in local storage. Its signed certificate is stored in a protected cookie and the verified origin is derived from certificate claims server-side.

## Browser storage inventory

| Name | Storage | Purpose |
| --- | --- | --- |
| `pinchana-settings` | local storage | UI and download preferences |
| `pinchana_cookie_consent` | local storage | Hides the informational cookie banner after acceptance |
| `pinchana_web_session` | HttpOnly, SameSite=Strict cookie | Signed API browser-session token |
| `pinchana_instance` | HttpOnly, SameSite=Strict cookie | Project-signed certificate for the selected custom API |
| `pinchana_locale` | HttpOnly, SameSite=Lax cookie | Selected interface language, retained for one year |
| `cf_clearance` (conditional) | Cloudflare cookie | May exist only if Turnstile pre-clearance or a Cloudflare Challenge is enabled |

Session and instance cookies use `Secure` in production and path `/`. The instance cookie's `Max-Age` is the remaining certificate lifetime. Selecting or restoring an instance deletes the current session cookie; restoring default also deletes the instance cookie.

Pinchana Web uses no analytics, advertising, tracking, or marketing cookies. Font Awesome and application fonts are served from the same origin. The browser does not contact external font providers. Pinchana does not embed third-party platform login widgets or scripts, although processing a URL necessarily asks the API to contact that platform server-side.

## Interface languages

English and Ukrainian are active interface languages. The locale cookie keeps server-rendered and client-rendered content consistent. Legal catalogs have a separate approval list; when a selected interface language does not have approved legal text, the authoritative English document is displayed with a fallback notice.

Translation catalogs are reviewed as JSON pull requests. See the [translation guide](/translating/) for the `source` and `translation` format.

The About diagnostics view is generated on demand. It exposes the public Web revision, sanitized API build manifest, coarse browser/device feature flags, viewport class, and generic processing state. Its copy action omits source URLs, media titles, profile data, custom origins, IP addresses, secrets, and the full browser user agent.

Clearing site data removes preferences, consent state, selected custom instance, and verification session. Downloaded files are outside Pinchana storage and remain wherever the browser saved them.
