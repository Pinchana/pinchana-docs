## Requirements and commands

Use Bun 1.3.14 or the version recorded by `packageManager`. Next.js runs through Bun; a separate global Node.js installation is not required for the documented commands.

```sh
git clone https://github.com/Pinchana/pinchana-web.git
cd pinchana-web
cp .env.example .env.local
bun install --frozen-lockfile
bun run dev
```

Before a change is ready:

```sh
bun run test
bun run lint
bunx tsc --noEmit
bun run build
```

`postinstall`, `predev`, and `prebuild` synchronize FFmpeg assets. Run `bun run sync:ffmpeg` explicitly when diagnosing missing assets.

## Environment variables

| Variable | Visibility | Required | Meaning |
| --- | --- | ---: | --- |
| `PINCHANA_API_URL` | server only | yes | Default API origin, such as `http://localhost:8080` locally or `https://api.pinchana.cc` in production |
| `NEXT_PUBLIC_TURNSTILE_SITE_KEY` | browser/public | yes | Widget site key registered for the Web hostname; an official always-pass test key may be used locally |
| `NEXT_PUBLIC_PINCHANA_WEB_COMMIT` | browser/public | production diagnostics | Full hexadecimal Web revision baked at build time; invalid or absent values display as `development` |
| `PINCHANA_INSTANCE_PUBLIC_KEY` | server only | for custom origins | Ed25519 public PEM used to verify project-issued instance certificates; use `\n` in one-line environment UIs |

The API separately needs the matching Turnstile private secret, expected Web hostname/action, and HMAC session secret. The Web app never receives those values.

## Project map

| Path | Responsibility |
| --- | --- |
| `app/Home.tsx` | Gate, URL form, results, settings, downloads, and notifications |
| `app/api/*` | Instance, verify, session, scrape, capabilities, DLP, and media BFF routes |
| `lib/pinchana.ts` | Cookies, origin selection, certificate verification, error mapping, URL rewriting |
| `app/lib/audio-converter.ts` | Local FFmpeg load and MP3 conversion |
| `scripts/sync-ffmpeg-core.mjs` | Copies static core and worker assets |
| `messages/app`, `messages/legal`, `i18n` | Paired source/translation catalogs, locale selection, and legal approval gates |
| `app/policy`, `app/usage` | Legal/privacy pages |

<Aside type="caution">
Static export is unsupported. Route handlers, server-only variables, cryptographic verification, streaming proxies, and HttpOnly cookies require a running Next.js server.
</Aside>

Common local failures are a missing API URL, mismatched site/secret Turnstile pair, hostname/action mismatch, a session secret shorter than 32 characters, unavailable API identity response, missing synchronized FFmpeg assets, or DLP controls hidden because the API capability is disabled or unhealthy.

Local success with Cloudflare test keys does not validate production challenge behavior. In production, first determine whether the browser sends `POST /api/verify`. If it does not, investigate the widget and browser environment before changing session code. Error `600010`, Private Access Token failures, extension `content.js` policy messages, or WebGL context loss are browser or challenge signals rather than evidence of a Pinchana session failure.
